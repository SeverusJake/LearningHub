# Playbook — POD Merch

## Phase 1 — Setup

1. **[YOU]** Create a Redbubble account and a TeePublic account, complete each platform's seller/payout onboarding, and confirm PayPal is selected and active as the payout method on both — see README.md's eligibility check before doing anything else.
2. **[AI]** Drafts a short artist bio/profile description (2-4 honest sentences, no invented backstory) for both platforms' public profile pages: your name or handle, one line on the kind of designs you make (e.g. "text-first designs for [the niches you're targeting]"), and a line inviting follows/requests. **[YOU]** edits it so it actually sounds like you and posts it on both profiles.
3. **[YOU]** Set up TRACKER.md as your running log before the first upload — one row per work session from day one, so Phase 3 scaling decisions have real data behind them instead of a gut feeling.

## Phase 2 — Operating loop (burst: 2-3 hour session producing 10-20 designs)

1. **Niche research (15-20 min).** **[AI]** researches candidate niches using Google Trends (trends.google.com, free) for relative interest over time, plus each platform's own search-autocomplete: type a broad niche term into Redbubble's and TeePublic's on-site search boxes and read what autocomplete suggests — those are real buyer queries, not guesses. Keep a running niche scoreboard the same way money/05 keeps one for its own niche picks: a table of niche name, Trends signal, autocomplete hits, competition eyeballed (search the niche + "shirt" and see what's already there), and a 1-5 score — so a niche you've already researched never gets re-scored from scratch next burst.

2. **Design batch — text-based first.** For a beginner catalog, text-based designs (clean typography, no complex illustration) sell more reliably and trademark-check faster than graphic art. **[AI]** drafts designs against a real formula per niche, for example:
   - **Profession pride:** "[Profession]. Not by choice, by [trait]." → "Nurse. Not by choice, by stubbornness."
   - **Hobby pun:** a play on the hobby's own vocabulary → "Dungeon Master by Night, [ordinary day job] by Day" or "Let's Taco 'Bout Pickleball."
   - **Identity/pride statement:** "[Group] [simple bold claim]" → "Electricians Keep the Lights On."
   - **Inside-joke shorthand:** a phrase specific enough that only that niche's audience recognizes and wants it, e.g. "Powered by Coffee and Overtime" for a shift-work niche — original wording only, checked the same as everything else, never a co-opted slogan someone else already owns.

   Every design gets exported as a transparent-background SVG or a PNG at 4500x5400px (the standard high-res print size both platforms recommend) before it goes anywhere near an upload screen.

3. **Design batch — graphic designs second.** Once the text batch for a niche is queued, **[AI]** drafts 1-2 simple original graphic concepts for the same niche (an icon, a small illustrated scene). Keep these simple — complex illustration takes longer to make and trademark-check, and it doesn't outsell text designs in a beginner catalog.

4. **[YOU] trademark-checks every single phrase before it goes near an upload button. This step is logged in TRACKER.md and is non-negotiable.** For each phrase: run it through USPTO TESS (tmsearch.uspto.gov) for an exact and a sound-alike search, then check it against Redbubble's and TeePublic's current IP/content policy pages. Log every check in TRACKER.md's trademark-check log (format below) before uploading — a phrase with no logged check does not go up, no matter how obviously fine it seems.

5. **Upload with AI-written titles/tags/descriptions.** **[AI]** writes the title, tags, and description for each design, respecting each platform's current character/tag limits (both publish these on their own upload pages, and both have changed them before — confirm the live limit rather than trusting an old number). Use the title/tag skeletons below as the starting formula, adapted to whatever the platform's live upload form actually allows that day.

6. **Enable a sensible product range per design.** Turn on shirts (multiple cuts/colors), stickers, and mugs as the default core range for every design — broad-appeal, high-margin formats that don't need per-product art rework. Skip formats that need major layout changes (phone cases, all-over-print, etc.) until a design is already proven.

7. **Weekly: review favorites/views/sales.** **[YOU]** spends 15-20 minutes in each platform's own stats dashboard, updates the niche scoreboard with what actually moved, and lets that decide next burst's niche picks — favorites and real sales are the signal, views alone are not.

## Phase 3 — Scaling

A niche that clears README.md's scale bar (repeat sales, not a fluke) gets built out to 20 design variations — the same core joke or angle across different color-ways, alternate phrasings, and layout treatments — so its buyers have real choice instead of one design. Time new uploads to land 6-8 weeks ahead of that niche's seasonal peak, since search interest and buying intent both ramp before the actual date. Example calendar (a starting point to adapt against your own Trends data, not a fixed rule):

| Niche | Peak season | Upload by |
|---|---|---|
| Teacher/school | back-to-school | early July |
| Nurse/healthcare | Nurses Week (May) | mid-March |
| Dad/Father's Day | Father's Day (June) | late April |
| Mom/Mother's Day | Mother's Day (May) | late March |
| Christmas/holiday | December | mid-October |
| Fitness/New Year | January resolutions | early December |
| Halloween | October | mid-August |
| Graduation | May-June | mid-March |

Consider a third POD platform (Society6, Threadless, Merch by Amazon, etc.) only after independently reading that platform's own current IP and AI-disclosure policy end to end — don't extend this folder's compliance floor by assumption, since each platform writes its own rules and Merch by Amazon in particular runs its own separate approval/tier system worth checking before planning around it.

## Templates and prompts

### Design-brief formula

> **Niche:** [specific profession/hobby/identity — never "animals" or "nature"]
> **Angle:** [pun / pride statement / inside joke / profession-specific complaint — pick one]
> **Format:** text-only, [font style: bold sans / hand-lettered / retro varsity], 1-2 colors max
> **Core line:** [the exact phrase — this is what gets trademark-checked]
> **Product fit:** shirt + sticker + mug (default range)
> **Reference note:** [why this niche/angle — e.g. "autocomplete showed 'dog groomer shirt' and 'dog groomer gift' as top suggestions"]

### Title/tag skeleton — Redbubble

> **Title:** [Core phrase] | [Niche] Gift — (confirm the current character limit on Redbubble's own upload page before finalizing)
> **Tags:** niche noun, niche noun plural, occasion (birthday/christmas/etc), "gift for [niche]", style descriptor (funny/cute/vintage), product-adjacent terms (shirt, mug, sticker) — (confirm the current tag-count limit on-platform)

### Title/tag skeleton — TeePublic

> **Title:** [Core phrase] — Funny [Niche] Gift Idea — (confirm the current character limit on TeePublic's own upload page before finalizing)
> **Tags:** same pattern as Redbubble, but prioritize the 5-8 highest-signal terms first — (confirm the current tag-count limit on-platform, TeePublic typically allows fewer than Redbubble)
